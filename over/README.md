## Invest
I - independente, uma boa história de usuário nao depende de outra, pois são difíceis de estimar e priorizar. Caso a história for grande, ela é um épico.
N - negociável, Um ponto de partida para uma conversa ou uma abertura para que a equipe sugira soluções. Negocie, pois as coisas mudam: prioridade, ordem , reescrita ou descarte
V - de grande valor, se você não desrever o valor que o cliente terá, ela não serve para nada. Tem que ter divertimento e valor.
E - Todas estórias devem ter informação suficiente ou definição para permitir que a equipe faça uma estimativa sobre a história do usuário
S - Tem que ser pequena (small), ser do tamanho de uma sprint. Qualquer história de usuário maior do que isso será difícil de se planejar.
T - testável.
